Game Development

What game development projects should I do?

  1. 2D Platformer
    • Create a 2D platformer with: levels, enemies, and collectibles.
      • Focus on character movement, collision detection and level design.
      • Do not focus on graphics. Use an asset pack or simple shapes.
  2. Top-Down Shooter
    • Create a 2D top with: levels, enemies, and collectibles.
      • Focus on enemies with basic AI, weapon switching, and a scoring system.

What game engine do you recommend?

My person choice for indie game developers who are not interested in engine development would be the Godot Engine.

How would I get started making a game engine?

Personally, if your goal is to make a game, then use an engine. There is no sense in reinventing the wheel if you objective is to make a game. However, if your goal is to deeply understand how games function then I would first check out RayLib and write a C++ framework using it. This is a good introduction into making games without an editor.

Once you have completed that, then you can write the RayLib functions yourself. This would involve doing graphics programming with OpenGL, creating a window with GLFW, and creating or linking a physics engine into the project.

Only begin creating a game engine once you have created a game in a pre-existing engine, like Godot Engine.

Beginner Projects

  1. To-Do List Application: Create a simple terminal app that allows useres to add, edit, and delete tasks. Make sure you store tasks in a local database or file.
  2. Calculator: Create a simple terminal calculator that reads in input from the standard input stream and have it compute the result.
